# /cft:spec-create のガイダンスメッセージを日本語に変更


## 1. 背景と目的

### 背景

現在、/cft:spec-create のガイダンスメッセージが英語で表示されるので、日本語に変更する。

### 目的

日本語ユーザー向けに、/cft:spec-create コマンドの成功メッセージおよびガイダンスを日本語化し、より分かりやすい操作体験を提供する。
---

## 2. 対象ユーザー

- cc-craft-kit を日本語環境で使用する開発者
- 仕様駆動開発（SDD）ワークフローを実践するチーム
---

## 3. 受け入れ基準

### 必須要件

- [ ] /cft:spec-create 実行後の成功メッセージが日本語で表示される
- [ ] Next steps のガイダンスが日本語で表示される

### 機能要件

- [ ] 既存の英語メッセージを日本語に置き換える
- [ ] 他のコマンド（/cft:spec-get など）との整合性を保つ

### 非機能要件

- [ ] 日本語表記は自然で分かりやすい文言とする
- [ ] 技術用語（Spec ID、Phase など）は英語のまま維持可能
---

## 4. 制約条件

- プロンプトファースト原則に従い、スラッシュコマンドの .md ファイルを編集する
- 他の /cft:* コマンドのメッセージ形式と統一感を保つ
---

## 5. 依存関係

- `src/slash-commands/cft/spec-create.md` - 対象のスラッシュコマンド定義ファイル
---

## 6. 参考情報

- 他の日本語化済みコマンドのメッセージ形式を参考にする
---

## 7. 設計詳細

### 7.1 アーキテクチャ設計

#### 全体構成

```
src/slash-commands/spec-create.md  ← 修正対象（プロンプトベース設計書）
         │
         ▼ npm run sync:dogfood
.cc-craft-kit/slash-commands/spec-create.md  ← 同期先（実際に実行されるファイル）
```

#### 設計方針

- **プロンプトファースト原則**に従い、スクリプト変更なしでプロンプト（.md）のみを編集
- 既存の日本語化済みコマンド（spec-delete, spec-phase）のメッセージ形式を踏襲
- 用語は他コマンドと統一（「仕様書」「フェーズ」「ブランチ」など）

#### 処理フロー（変更なし）

```
/cft:spec-create 実行
       │
       ▼
Step 1-9: 既存処理（変更なし）
       │
       ▼
Step 10: 成功メッセージ表示 ← ここのテキストを日本語に変更
```
