## 7. 設計詳細

### 7.1 アーキテクチャ設計

#### 全体構成

本仕様はドキュメント整備タスクのため、コードの変更は伴わない。対象ファイルは以下の通り:

```
cc-craft-kit/
├── CHANGELOG.md          # リリースリンク追加
├── README.md             # コマンド形式修正
└── docs/
    ├── ARCHITECTURE.md   # バージョン・コマンド表記修正
    ├── QUICK_START.md    # コマンド形式修正
    └── GITHUB_PROJECTS.md # フェーズ情報修正
```

#### 設計方針

- ドキュメントのみの変更のため、アーキテクチャ変更なし
- 既存のフォーマット・スタイルを踏襲
- 統合コマンド形式 `/cft:command subcommand` に統一
- 4 フェーズモデル（requirements → design → implementation → completed）を徹底

### 7.2 実装方針

#### 修正対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| CHANGELOG.md | v0.1.4, v0.1.3 のリリースリンクを追加 |
| README.md | 古いコマンド形式を統合形式に更新、`/cft:db-info` を削除 |
| docs/ARCHITECTURE.md | バージョンを 0.1.4 に更新、コマンド表記を統合形式に修正 |
| docs/QUICK_START.md | 古いコマンド形式を統合形式に更新、`/cft:db-info` を削除 |
| docs/GITHUB_PROJECTS.md | フェーズ マッピングテーブルを 4 フェーズモデルに修正 |

#### 修正パターン

**コマンド形式の統一:**
- `/cft:task-list` → `/cft:task list`
- `/cft:task-start` → `/cft:task start`
- `/cft:task-done` → `/cft:task done`
- `/cft:knowledge-progress` → `/cft:knowledge progress`
- `/cft:knowledge-error` → `/cft:knowledge error`
- `/cft:knowledge-tip` → `/cft:knowledge tip`
- `/cft:sync-check` → `/cft:sync check`
- `/cft:sync-repair` → `/cft:sync repair`
- `/cft:db-info` → 削除（存在しないコマンド）
