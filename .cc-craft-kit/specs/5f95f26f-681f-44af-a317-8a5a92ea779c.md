## 7. 設計詳細

### 7.1 アーキテクチャ設計

#### 全体構成

v0.1.1 リリースはドキュメント整備とバージョン更新が主な作業であり、コード変更は最小限。

```
更新対象ファイル構成:
├── package.json           # バージョン番号更新
├── CHANGELOG.md           # 新規セクション追加
├── README.md              # コマンド一覧拡充
└── docs/
    ├── ARCHITECTURE.md    # バージョン・日付更新
    └── QUICK_START.md     # 高度なコマンド追加
```

#### 設計方針

1. **既存構造の維持**: ドキュメントの構造を変更せず、内容を拡充
2. **段階的更新**: バージョン更新 → CHANGELOG → README → docs の順で更新
3. **整合性確保**: 全ドキュメント間でバージョン・機能説明を統一

### 7.2 実装方針

#### 修正対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| package.json | `"version": "0.1.0"` → `"version": "0.1.1"` |
| CHANGELOG.md | `## [0.1.1] - 2025-11-27` セクション追加 |
| docs/ARCHITECTURE.md | バージョン 0.1.1、最終更新日を更新 |
| README.md | 全コマンド一覧セクション追加（36個） |
| docs/QUICK_START.md | 高度なコマンドセクション追加 |

#### 変更しないファイル

- src/ 配下のソースコード（機能変更なし）
- .cc-craft-kit/ 配下の設定ファイル（同期不要）
- tests/ 配下のテストファイル

### 7.3 機能マッピング

| 機能 | 現在の実装 | 新しい実装 | 実装場所 |
|-----|----------|----------|---------|
| バージョン表記 | 0.1.0 | 0.1.1 | package.json, ARCHITECTURE.md |
| コマンド一覧 | 13個記載 | 36個記載 | README.md |
| 変更履歴 | 0.1.0まで | 0.1.1追加 | CHANGELOG.md |
| 高度なコマンド説明 | なし | 追加 | QUICK_START.md |

### 7.4 テスト戦略

| テスト対象 | テスト内容 | 検証方法 |
|----------|----------|---------|
| ドキュメントリンク | リンク切れなし | 手動確認 |
| バージョン整合性 | 全ファイルで 0.1.1 | grep で確認 |
| コマンド一覧 | 実装と記載の一致 | スラッシュコマンドファイルと照合 |
| 日本語表現 | 統一されている | textlint で確認 |

### 7.5 移行計画

#### Phase 1: バージョン更新
