## 1. 背景と目的

### 背景

Projects を作成するとデフォルトでテーブルビューが追加されているが、効果的に使うためにはビューのカスタマイズが必須なので、それを分かりやすくドキュメントで案内する。

### 目的

GitHub Projects v2 を効果的に活用するための推奨ビュー設定をドキュメント化し、ユーザーがプロジェクト管理を最大限に活用できるようにする。

---

## 2. 対象ユーザー

- cc-craft-kit を使って GitHub Projects v2 でタスク管理を行う開発者
- Projects のビューカスタマイズ方法がわからないユーザー
- より効率的なプロジェクト管理方法を探しているユーザー

---

## 3. 受け入れ基準

### 必須要件

- [ ] 3 つの推奨ビュー（Board、Table、Roadmap）の設定方法をドキュメント化
- [ ] 各ビューの用途と利点を明記
- [ ] ステップバイステップの設定手順を記載

### 機能要件

- [ ] Board ビュー: Status フィールドによるカンバンボード設定
- [ ] Table ビュー: フィルター・ソート・グループ化の推奨設定
- [ ] Roadmap ビュー: 開始日・期日によるタイムライン表示設定
- [ ] 各ビューのスクリーンショット（またはテキストベースの説明）

### 非機能要件

- [ ] 初心者でも理解しやすい平易な説明
- [ ] QUICK_START.md または docs/GITHUB_PROJECTS.md に統合
- [ ] Markdown フォーマットで記述

---

## 4. 制約条件

- GitHub Projects v2 の UI は頻繁に更新されるため、UI 要素名は変更される可能性がある
- スクリーンショットはメンテナンスコストが高いため、テキストベースの説明を優先
- CLI からのビュー設定は GraphQL API で可能だが、初期設定は Web UI を推奨

## 6. 推奨ビュー設定（Design）

### 6.1 Board ビュー（カンバンスタイル）

**用途**: タスクの進捗を視覚的に管理

**設定手順**:

1. Project ページで「+ New view」をクリック
2. 「Board」を選択
3. ビュー名を「Status Board」に設定
4. 「Group by」で「Status」フィールドを選択
5. 列の順序を調整: Todo → In Progress → Done

**推奨カスタマイズ**:

- フィルター: `is:open` で未クローズの Issue のみ表示
- ソート: Priority（高→低）または Created date（新→古）
- 表示フィールド: Title, Assignees, Labels, Phase

**利点**:

- 一目で各ステータスのタスク数がわかる
- ドラッグ&ドロップでステータス変更可能
- WIP（Work In Progress）制限を視覚的に管理しやすい

---

### 6.2 Table ビュー（詳細管理）

**用途**: 全タスクの詳細情報を一覧表示・フィルタリング

**設定手順**:

1. デフォルトの「Table」ビューを使用（または新規作成）
2. ビュー名を「All Tasks」に設定
3. 表示する列を追加: Status, Phase, Assignees, Labels, Created, Updated

**推奨カスタマイズ**:

- **グループ化**: Phase でグループ化（requirements/design/tasks/implementation/completed）
- **ソート**: Priority（高→低）→ Created date（新→古）
- **フィルター例**:
  - 自分のタスクのみ: `assignee:@me`
  - 特定フェーズ: `phase:implementation`
  - 期日が近いタスク: `due:<7days`

**表示推奨フィールド**:

1. Title（必須）
2. Status（Todo/In Progress/Done）
3. Phase（requirements/design/tasks/implementation/completed）
4. Assignees
5. Labels（`phase:*` ラベル）
6. Created date
7. Updated date

**利点**:

- 詳細な情報を一覧で確認可能
- 強力なフィルター・ソート機能
- CSV エクスポート可能

---

### 6.3 Roadmap ビュー（タイムライン）

**用途**: タスクのスケジュールをタイムラインで可視化

**設定手順**:

1. Project ページで「+ New view」をクリック
2. 「Roadmap」を選択
3. ビュー名を「Timeline」に設定
4. 「Start date」と「End date」フィールドを設定（Issue の Milestone または Custom field）

**推奨カスタマイズ**:

- **グループ化**: Phase または Assignees
- **ズームレベル**: Month または Quarter
- **フィルター**: `is:open` で未完了タスクのみ

**注意事項**:

- Roadmap ビューを使用するには、各 Issue に開始日・期日を設定する必要がある
- cc-craft-kit では現在、開始日・期日のフィールドは自動設定されないため、手動で追加

**利点**:

- プロジェクト全体のスケジュールを把握
- タスク間の依存関係を視覚化
- リリース計画の策定に有用

---

### 6.4 その他の推奨設定

**Phase フィルタービュー**:
各フェーズごとに専用ビューを作成すると便利です。

- **Requirements View**: `phase:requirements` でフィルター
- **Implementation View**: `phase:implementation` でフィルター
- **Completed View**: `phase:completed` でフィルター

**優先度ビュー**:
Priority フィールドを追加している場合、優先度別のビューも有効です。

---

## 7. cc-craft-kit プロジェクトでの実践例

cc-craft-kit 開発プロジェクト自身では、以下のビュー構成を推奨します:

1. **Status Board** (Board): 日々の作業管理用
2. **All Tasks** (Table): 全タスクの詳細確認・検索用
3. **Requirements** (Table): requirements フェーズの Issue のみ表示
4. **In Progress** (Table): Status が "In Progress" の Issue のみ表示
5. **Timeline** (Roadmap): リリース計画用（オプション）
