## 1. 背景と目的

### 背景

Takumi（匠）は、類似商標が多々あるため、cc-xxx-kit という形式の新たな名称を考えたい（cc は、 Claude Code の略）。この開発キットの性質や特徴を表す xxx の部分を検討する。また、カスタムスラッシュコマンド名の /xyz の部分は単純に xxx を当てはめるのではなく、タイピング文字数少な目で xxx を短縮したものとしたい。例えば、現在の Takumi（匠）の場合、/cft:xxxx ではなく /tak:xxx のような感じが好ましい。

### 目的

1. 商標競合リスクを回避するため、cc-xxx-kit 形式の新名称を決定する
2. プロジェクトの性質・特徴を適切に表現する xxx 部分を選定する
3. タイピング効率を考慮した短縮スラッシュコマンド名を策定する
4. 名称変更に伴う影響範囲を特定し、移行計画を立案する

### 7.7 最終推奨案

**推奨名称**: **cc-craft-kit**

**推奨短縮コマンド**: `/cft:`

**理由**:

1. **高評価**: 87/100 点（cc-sdd-kit と同点トップ）
2. **商標リスク**: 低リスク
3. **Takumi との連続性**: "craft"（技巧）は "Takumi"（匠）の意味と親和性が高い
4. **国際性**: 英語圏のユーザーにも理解しやすい
5. **覚えやすさ**: Software Craftsmanship の文化と一致

**次点**: **cc-sdd-kit** (`/sdd:`)

- 商標リスクは最低だが、覚えやすさと国際性で劣る
- 技術者向けプロジェクトであれば、SDD の略語は許容範囲
