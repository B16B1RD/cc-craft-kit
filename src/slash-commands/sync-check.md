# 仕様書ファイルとデータベース間の整合性チェック

仕様書ファイル (`.cc-craft-kit/specs/*.md`) とデータベース (`specs` テーブル) 間の整合性をチェックします。

## 実行内容

1. specs ディレクトリから全`.md`ファイルを取得
2. データベースから全`specs`レコードを取得
3. IntegrityChecker.check() で差分を検出
4. レポートをコンソールに出力

## 出力内容

- **Total Files**: ファイルシステム上の仕様書ファイル総数
- **Total DB Records**: データベース上の仕様書レコード総数
- **Synced**: 正常に同期済みの仕様書数
- **Files Only (DB未登録)**: ファイルのみ存在し、データベースに未登録の仕様書
- **DB Only (ファイル削除済み)**: データベースのみ存在し、ファイルが削除済みの仕様書
- **Mismatch (メタデータ不一致)**: ファイルとデータベースでメタデータが一致しない仕様書
- **Sync Rate**: 同期率 (%)

## 使用例

```bash
/cft:sync-check
```

---

以下のコマンドを実行して整合性をチェックしてください。

```bash
npx tsx .cc-craft-kit/commands/sync/check.ts
```

チェック結果に基づいて、修復が必要な場合は以下のコマンドを実行してください。

```bash
/cft:sync-repair
```
